## What happened

DeepBook appears to have performed an oracle rollover around:

```text
2026-06-10 14:00 UTC
```

creating replacement oracles for future expiries.

Old oracles were left:

```text
status = active
```

but stopped receiving:

```text
prices/latest
svi/latest
```

updates.

So:

```text
status != live
```

This is the key lesson.

## Your production filter

Don't use:

```js
oracle.status === "active"
```

Use:

```js
const lastUpdate = Math.max(
  svi?.checkpoint_timestamp_ms || 0,
  price?.checkpoint_timestamp_ms || 0
);

const isLive =
  Date.now() - lastUpdate <
  5 * 60 * 1000;
```

Then:

```js
const liveOracles = active.filter(
  o => o.isLive
);
```

## One more improvement

Instead of freshness alone, deduplicate by expiry:

```js
const byExpiry = new Map();

for (const oracle of liveOracles) {
  const expiry = oracle.expiry;

  if (!byExpiry.has(expiry)) {
    byExpiry.set(expiry, oracle);
    continue;
  }

  const existing = byExpiry.get(expiry);

  if (
    oracle.lastUpdate >
    existing.lastUpdate
  ) {
    byExpiry.set(expiry, oracle);
  }
}

const canonicalOracles =
  [...byExpiry.values()];
```

This protects you if DeepBook does another rollover in the future.
